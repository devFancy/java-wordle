# Wordle 미션

## 1. 게임 실행 프로세스

동작 순서: 초기화 -> 게임 시작 -> 정답 단어 선정 (계산) -> 입력 -> 정답 여부 확인

- **게임 준비 단계**
  - 데이터 로드: words.txt 파일을 읽어 단어 리스트(배열/리스트)를 메모리에 적재
  - 정답 결정:
    - 오늘의 날짜와 기준 날짜(2021년 6월 19일)의 차이를 계산
    - 차이값 % 리스트 크기 인덱스에 해당하는 단어를 오늘 정답으로 설정
  - 게임 시작 메시지: 환영 문구와 규칙을 출력

- **사용자 입력 및 검증 단계**
  - 단어 입력: 사용자로부터 단어를 입력받음
  - 유효성 검사:
    - 길이: 5글자
    - 형식: 영문 알파벳으로만 구성
    - 존재 여부: words.txt 목록에 포함된 단어
  - 재시도: 유효하지 않은 입력일 경우 에러 메시지를 출력하고 다시 입력을 받음

- **입력 받은 답안과 정답을 비교 및 상태 간직**
  - 입력 단어와 정답이 문자가 같고 위치가 일치: **초록(🟩)**
  - 입력 단어와 정답이 문자는 같지만 위치가 다름: **노란색(🟨)**
  - 입력 단어와 정답이 문자와 위치가 같지 않음: **회색(⬜)**

- 결과 누적 및 출력
  - 결과 누적: 현재 시도의 타일 결과(예: ⬜⬜🟨🟩⬜)를 이력에 저장
  - 전체 이력 출력: 지금까지 진행한 모든 시도의 타일 이력을 화면에 다시 출력
```text
⬜⬜🟨🟩⬜
🟨⬜⬜⬜🟩
```

- 종료 판정
  - 승리: 5글자 모두 초록색(🟩)인 경우, 시도 횟수(예: 4/6)를 출력하고 게임을 종료
  - 패배: 시도 횟수가 6회에 도달했는데 정답을 맞추지 못한 경우 게임을 종료
  - 진행: 종료 조건에 해당하지 않으면 다시 **[사용자 입력 및 검증 단계]**부터 반복

## 2. 상세 구현 리스트

1. 파일 유틸리티
- [x] words.txt 파일을 읽어오는 유틸리티 구현
- [x] 파일 읽기 실패 등 예외 상황 처리

2. 날짜 유틸리티
- [x] 일수 차이 계산: 오늘 날짜와 기준일 사이의 경과 일수 반환

3. 단어 사전
- [x] 단어 사전 초기화: 파일로부터 단어 목록을 불러오기
- [x] 단어 존재 여부 확인: 입력된 단어가 사전에 존재하는 유요한 단어인지 검증
- [ ] N 번째 단어를 반환

4. 심판
- [x] 타일 일치 여부 판별: 정답과 입력 단어를 비교하여 초록(🟩), 노랑(🟨), 회색(⬜) 타일을 반환
  - (선택) 중복 문자 처리: 이미 비교한 **답안 초록(🟩) 문자**는 또 비교하지 않도록 처리
- [ ] 결과 생성: 비교 결과를 저장할 데이터 구조 반환

5. 게임 진행자(=Wordle)
- [ ] 정답 단어 선정: 날짜 기반 인덱스를 기준으로 오늘의 정답 선정
  - 기준일 관리: 2021년 6월 19일이라는 기준 시점 정의
- [ ] 라운드 관리: 현재 몇 번째 시도인지 추적
- [ ] 기록 보관: 이전 라운드들의 타일 결과를 순서대로 저장
- [ ] 게임 종료 판정: 
  - 모든 글자가 초록인 경우 '승리'
  - 6회 시도 후 정답을 못 맞추면 '패배'

6. 입/출력
- [ ] 입력값 유효성 검증
  - [ ] 입력값이 5글자인지 확인
  - [ ] 입력값이 영문 소문자인지 확인
- [ ] 게임 현황 출력
- [ ] 최종 결과 메시지 출력

## 3. 페어 프로그래밍

- 각자 PR을 보내야 하므로, 아래의 **Git 원격 저장소 추가 방식**을 활용
  - 동료 레포지토리 연결
    - 본인의 로컬 터미널에서 실행: git remote add pair [https://github.com/페어계정/java-wordle.git](https://github.com/%ED%8E%98%EC%96%B4%EA%B3%84%EC%A0%95/java-wordle.git)
  - 페어 교대 시 코드 가져오기
    - 페어 B는 `git pull pair {페어 브랜치명}` 명령어로 페어 A의 작업 내역을 자신의 로컬로 가져온다.

- 30분 코딩, 10분 회고 (호흡 맞추기)
  - 30분 코딩 (Driver & Navigator)
    - **역할 교대:** 매 30분마다 반드시 Driver와 Navigator 역할을 바꿉니다.
      - `공통`: 서로의 싱크를 맞춰야 한다.
      - `드라이버`: 말하면서 코드를 작성한다.
      - `네비게이터`: 드라이버에게 시간을 주돼, 요구사항에서 벗어나면 조언을 말한다.
    - **생각 공유:** Driver는 지금 왜 이 코드를 치고 있는지 말로 설명하고 Navigator는 의문이 들면 즉시 질문합니다.
  - 10분 회고 (Retrospective)
    - KPT 회고지만 가볍게 진행
